Backend REST API Development with Spring Boot

A guide to coding standards for scalable, maintainable, and readable APIs

1. Project Structure

Adopt a clear and standardized layered architecture for your Spring Boot project:

        src/main/java
        └── com
            └── example
                └── app
                    ├── controller
                    ├── service
                    ├── repository
                    ├── model
                    └── dto

2. Code Naming Conventions

  • Class Names: Use PascalCase (e.g., UserController, ProductService).
  • Method Names: Use camelCase (e.g., findByName(), createOrder()).
  • Variable Names: Use camelCase for variables.
  • Constants: Use UPPER_SNAKE_CASE (e.g., MAX_RETRIES, DEFAULT_TIMEOUT).

3. Controller Layer

Define REST endpoints and handle HTTP requests in a controller class:

        @RestController
        @RequestMapping("/api/v1/users")
        public class UserController {
            @GetMapping("/{id}")
            public ResponseEntity getUserById(@PathVariable Long id) {
                // Implementation
            }
        }

4. Service Layer

Place business logic inside services and handle transactions:

        @Service
        @Transactional
        public class UserService {
            public UserDTO getUserById(Long id) {
                // Implementation
            }
        }

5. Repository Layer

Use Spring Data JPA to interact with the database:

        @Repository
        public interface UserRepository extends JpaRepository {
            Optional findByEmail(String email);
        }

6. Error Handling

Use @ControllerAdvice for global exception handling:

        @ControllerAdvice
        public class GlobalExceptionHandler {
            @ExceptionHandler(EntityNotFoundException.class)
            public ResponseEntity handleEntityNotFound(EntityNotFoundException ex) {
                return new ResponseEntity<>(new ErrorResponse("Not Found", ex.getMessage()), HttpStatus.NOT_FOUND);
            }
        }
